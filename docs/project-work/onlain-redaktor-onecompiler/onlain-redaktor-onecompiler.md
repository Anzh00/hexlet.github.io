# Онлайн-редактор OneCompiler для работы над проектом

> В некоторых проектах код можно писать и проверять прямо в браузере — без установки инструментов на свой компьютер. Для этого мы используем онлайн-редактор OneCompiler. В этой статье разберём, как он устроен и как с его помощью готовить решение проекта.

[OneCompiler](https://onecompiler.com/) — это бесплатный онлайн-редактор, который запускает код прямо в браузере. Он не требует регистрации и поддерживает много языков и баз данных. Это удобно, когда не хочется тратить время на установку и настройку окружения ради учебного проекта.

Сейчас мы используем OneCompiler в проектах по **SQL**, поэтому все примеры в статье — на SQL и базе данных **PostgreSQL**. Запросы выполняются на настоящей PostgreSQL — той же, что и автоматическая проверка проекта. Это значит, что запрос, который успешно отработал в OneCompiler, будет вести себя так же и при проверке решения.

## Открываем редактор

Для работы с SQL откройте страницу [onecompiler.com/postgresql](https://onecompiler.com/postgresql). Сразу откроется готовый к работе редактор с небольшим примером. Версия базы данных уже выбрана — PostgreSQL, её менять не нужно.

![Стартовый экран OneCompiler](/img/docs/img-192--onecompiler-start.png)

<!-- ANNOTATION: на этом скриншоте желательно подсветить рамкой/стрелкой три зоны:
     (1) поле редактора слева, (2) красную кнопку «Run» сверху по центру,
     (3) панель «Output» справа. И подпись «PostgreSQL уже выбран» к кнопке выбора языка вверху. -->

Окно разделено на две части:

- **Слева — редактор кода.** Здесь вы пишете SQL: создаёте таблицы, наполняете их данными и пишете запросы.
- **Справа — панель `Output`.** Сюда выводится результат после запуска.
- **Сверху — кнопка `Run`.** Она запускает весь код из редактора.

## Пишем и выполняем запрос

Удалите пример из редактора и напишите свой SQL. В одном окне можно держать всё сразу: создание таблицы, наполнение данными и сам запрос. После этого нажмите кнопку **Run** — результат появится в панели `Output` справа.

![Результат выполнения запроса в OneCompiler](/img/docs/img-193--onecompiler-result.png)

PostgreSQL выполнит команды по очереди и покажет результат последнего запроса в виде таблицы. Если в запросе есть ошибка, в панели `Output` появится сообщение с её описанием — это удобно: можно сразу исправить запрос и запустить снова.

## Как работать над проектом

В проекте каждый запрос — это отдельный файл в вашем репозитории. Поэтому работайте так:

1. Напишите в OneCompiler сначала создание таблицы и наполнение данными, а затем — нужный запрос.
2. Нажмите **Run** и убедитесь, что результат правильный.
3. Скопируйте **только сам запрос** (без создания и наполнения таблицы) в отдельный файл решения с именем из задания — например, `query.sql`.

Создание и наполнение таблицы держите в одном месте и переиспользуйте при проверке каждого следующего запроса: меняете внизу только сам запрос, снова нажимаете **Run** — и проверяете следующий файл.

:::tip
Регистрироваться и нажимать **Save** не обязательно. Сервис нужен только для того, чтобы проверить код. Готовое решение вы храните не в OneCompiler, а в своём репозитории на GitHub.
:::

## Сохраняем решение на GitHub

Когда запрос готов и проверен, его нужно загрузить в репозиторий проекта на GitHub. Делать это через командную строку и Git не обязательно — достаточно веб-интерфейса GitHub.

Откройте свой репозиторий проекта, нажмите **Add file** и выберите **Upload files**:

![Загрузка файлов в репозиторий через веб-интерфейс GitHub](/img/docs/img-194--onecompiler-github-upload.png)

Затем перетащите файлы с запросами в открывшееся поле или выберите их вручную и подтвердите загрузку (кнопка **Commit changes**). После этого файлы появятся в репозитории, и их сможет проверить автоматическая система.
