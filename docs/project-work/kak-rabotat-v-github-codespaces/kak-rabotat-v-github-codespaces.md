---
title: "Как работать в GitHub Codespaces"
---

GitHub Codespaces — это редактор кода прямо в браузере. Не нужно ничего устанавливать: открываешь репозиторий, запускаешь Codespace и сразу пишешь код.

---

## Шаг 1. Открыть репозиторий

Перейди по ссылке на репозиторий проекта в GitHub. Ты увидишь список файлов и кнопку **Code** в правой части страницы.

![Страница репозитория](/img/docs/img-179--codespaces-repo.png)

---

## Шаг 2. Создать Codespace

Нажми зелёную кнопку **Code**. Откроется выпадающее меню с вкладкой **Local** по умолчанию.

![Меню Code](/img/docs/img-180--codespaces-code-button.png)

Переключись на вкладку **Codespaces**.

![Вкладка Codespaces](/img/docs/img-181--codespaces-tab.png)

Нажми **Create a codespace on main**. GitHub начнёт готовить окружение — это занимает около 30–60 секунд.

![Создание Codespace](/img/docs/img-182--codespaces-creating.png)

После загрузки откроется редактор VS Code прямо в браузере.

---

## Шаг 3. Интерфейс редактора

Ты увидишь три основные области:

- **Слева** — Explorer (проводник с файлами проекта)
- **В центре** — редактор кода
- **Снизу** — Terminal (терминал для запуска команд)

![Интерфейс VS Code](/img/docs/img-183--codespaces-vscode-interface.png)

---

## Шаг 4. Создать файл

Наведи курсор на название проекта в Explorer — появятся иконки. Нажми **New File...** (иконка листа со знаком плюс).

![Кнопка создания файла](/img/docs/img-184--codespaces-new-file-button.png)

Введи имя файла и нажми **Enter**.

![Ввод имени файла](/img/docs/img-185--codespaces-new-file-input.png)

Файл создан и сразу открыт в редакторе.

![Файл открыт в редакторе](/img/docs/img-186--codespaces-file-created.png)

---

## Шаг 5. Написать и сохранить код

Напиши код в открытом файле. Чтобы сохранить изменения, нажми **Ctrl+S** (или **Cmd+S** на Mac).

Сохранённый файл теряет точку (●) рядом с именем в заголовке вкладки.

![Код написан и сохранён](/img/docs/img-187--codespaces-code-written.png)

---

## Шаг 6. Запустить код

Кликни в область терминала в нижней части экрана (надпись **TERMINAL**).

Введи команду для запуска и нажми **Enter**. Команда зависит от языка проекта — имя файла смотри в задании:

| Язык       | Команда                              |
|------------|--------------------------------------|
| Python     | `python <файл>.py`                   |
| JavaScript | `node <файл>.js`                     |
| PHP        | `php <файл>.php`                     |
| Go         | `go run <файл>.go`                   |
| Java       | `javac <Файл>.java && java <Файл>`   |
| C#         | `dotnet run`                         |

Результат появится в терминале сразу после команды.

![Запуск кода в терминале](/img/docs/img-188--codespaces-run-output.png)

---

## Шаг 7. Отправить изменения на GitHub

Нажми на иконку **Source Control** в левой панели (три кружка со стрелками) — там будет значок с цифрой, обозначающей количество изменённых файлов.

В поле **Message** напиши краткое описание того, что ты сделал.

![Поле сообщения коммита](/img/docs/img-189--codespaces-commit-message.png)

Нажми на стрелку рядом с кнопкой **Commit** и выбери **Commit & Push**.

![Меню Commit & Push](/img/docs/img-190--codespaces-commit-push-menu.png)

Если появится вопрос «Would you like to stage all your changes and commit them directly?» — нажми **Always**.

После этого изменения отправятся в GitHub. В разделе **GRAPH** ты увидишь свой коммит в списке.

![Изменения отправлены](/img/docs/img-191--codespaces-pushed.png)

Готово! Твои изменения сохранены в GitHub.
